### Key Decisions
1. **GF(256) via log/antilog:** Doubled EXP table so `EXP[LOG[a]+LOG[b]]` never overflows; mul/div/inverse all guard zero.
2. **Reed–Solomon by LFSR:** Synthetic long division (O(data×ecc)); generator poly folded from `(x − 2ⁱ)` factors; Wikiversity vector validates.
3. **Smallest-fitting version:** Dynamic version selection; char-count widths per spec; terminator + `0xEC/0x11` padding.
4. **Block interleaving:** Data split across blocks, RS per block, column-major interleave to mitigate burst damage.
5. **Matrix layout:** Finders/separators/timing (1×1 grid), alignment (corner), format/version info (BCH(15,5) and (18,6)), 2-wide zig-zag.
6. **8-pattern masking:** All 4 penalty rules; choose best; BCH format bits encode which mask was used.
7. **Decoder choice:** Prefer `pyzbar` (zbar) — reliable on tiny symbols. OpenCV fallback only; skip tests if absent.

This challenge definitively establishes:
1. **Python package structure** — one module per stage, `__main__.py`, `.venv` + artifacts gitignored.

3. **Decoder selection hygiene** — zbar preferred; flaky decoders (e.g., OpenCV on tiny symbols) skip tests gracefully.
