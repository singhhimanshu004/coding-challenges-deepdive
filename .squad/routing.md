# Routing Rules

## Default Routes
- **Challenge planning, architecture, language selection** → Grant 🏗️
- **Challenge solutions, code, documentation** → Malcolm 🔧
- **Code review, testing, completeness** → Ellie 🧪
- **Session logs, decisions, memory** → Scribe 📋
- **Work queue, backlog, status** → Ralph 🔄

## Category Routing

### Unix/CLI Tools (wc, grep, sed, jq, curl, cut, sort, uniq, head, tail)
- **Primary:** Malcolm 🔧 — writes solution code and docs
- **Review:** Grant 🏗️ — reviews architecture and Unix semantics

### Networking & Protocols (DNS resolver, traceroute, port scanner, NTP client, IRC client)
- **Primary:** Malcolm 🔧 — implements protocol handling and networking code
- **Review:** Grant 🏗️ — reviews protocol correctness and edge cases

### Servers & Infrastructure (web server, load balancer, Redis, Memcached, NATS)
- **Primary:** Malcolm 🔧 — builds server implementations
- **Review:** Grant 🏗️ — reviews architecture decisions, concurrency model

### Games (Pong, Tetris, Chess, Snake, Minesweeper, Game of Life)
- **Primary:** Malcolm 🔧 — implements game logic and rendering
- **Review:** Ellie 🧪 — reviews game logic correctness, edge cases

### Data & Compression (JSON parser, Huffman compression, Bloom filter, Base64)
- **Primary:** Malcolm 🔧 — implements algorithms and data structures
- **Review:** Grant 🏗️ — reviews algorithm correctness and performance

### Applications (URL shortener, chat server, shell, Pastebin, Dropbox)
- **Primary:** Malcolm 🔧 — full implementation
- **Review:** Full team — Grant (architecture), Ellie (testing), for complex multi-component challenges

### Developer Tools (Git, Docker, rate limiter, load tester)
- **Primary:** Malcolm 🔧 — implements tool internals
- **Review:** Grant 🏗️ — reviews design decisions and correctness

## Escalation
- Multi-language decisions → Grant
- Quality disputes → Ellie
- Complex challenges requiring multiple components → full team review
