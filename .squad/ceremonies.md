# Ceremonies

## Challenge Review
- **When:** Before starting a new challenge category
- **Who:** Grant (leads), Malcolm, Ellie
- **Purpose:** Review upcoming challenges in the category, decide language choices, identify shared patterns, set priority order
- **Output:** Updated backlog with prioritized challenges and language assignments

## Category Retro
- **When:** After completing all challenges in a category
- **Who:** Full team
- **Purpose:** Review what went well, what could improve, patterns learned, reusable code identified
- **Output:** Decision log entry, lessons learned, any refactoring tasks

## Solution Quality Gate
- **When:** Before committing a completed challenge solution
- **Who:** Grant (architecture), Ellie (code review)
- **Purpose:** Ensure solution meets quality bar — correct, well-documented, tested, educational
- **Output:** Approved solution or revision notes
